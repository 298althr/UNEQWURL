# UNEQWURL Sound Lab

**UNEQWURL** is a mobile-first web app for training your ear on equalization. Pick a song, tune five live EQ sliders while the track plays, submit your mix, and get an instant score based on how close you are to the song’s benchmark profile.

The name comes from the signature fifth control — **UNEQWURL** — a peaking filter fixed at **298 Hz**, designed to shape low-mid warmth and punch without overlapping the standard low / mid / high bands.

---

## How it works

1. **Register / sign in** — username + password (no email).
2. **Select a song** — only tracks with a ready benchmark appear in the list.
3. **Open the EQ room** — sliders start at a **random mix** (Low, Mid, High, Gain randomized between −12 and +12 dB; **UNEQWURL always starts at 0**). Audio plays through a live Web Audio chain powered by [WEQ8](https://github.com/teropa/weq8).
4. **Adjust sliders** — find the best-sounding combination for that track.
5. **Submit mix** — your settings are scored against the song’s hidden **benchmark** (the ideal EQ profile generated by the analysis engine). A perfect match scores **100**.

Admins can review users, submissions, and edit benchmark values at `/admin`.

---

## The five sliders

All sliders range from **−12 dB to +12 dB**. Default is **0 dB** (no change). Positive values boost; negative values cut.

| Slider | Filter type | Center / corner freq | What it does |
|--------|-------------|----------------------|--------------|
| **Low** | Low shelf | 100 Hz | Shapes **bass and sub-bass**. Boost adds weight and rumble; cut thins out muddy low end. |
| **Mid** | Peaking (Q 0.7) | 1,000 Hz | Controls **lower-mid / body** — vocals, snare body, guitar warmth. Boost adds fullness; cut reduces honk and boxiness in the core midrange. |
| **High** | High shelf | 8,000 Hz | Shapes **brightness and air** — cymbals, hi-hats, vocal sibilance, sparkle. Boost opens the top end; cut dulls harsh or brittle treble. |
| **Gain** | Peaking (Q 0.7) | 4,000 Hz | **Presence / upper-mid clarity** — the “forward” part of the mix (vocal intelligibility, snare crack, mix punch). Distinct from High (shelf) and Mid (1 kHz). |
| **UNEQWURL** | Peaking (Q 1.4) | **298 Hz** | The **UNEQWURL signature band** — low-mid warmth and punch. Boost emphasizes kick/bass body and male vocal weight; cut reduces muddiness and boxiness in the 298 Hz region. Steps in **0.5 dB** increments. |

### Why UNEQWURL is separate

Typical “Low” shelves sit below ~100 Hz and “Mid” peaks around ~1 kHz. The **298 Hz** region sits between them — where kick drums, bass fundamentals, and vocal chest tone live — but isn’t fully controlled by either. **UNEQWURL** targets that gap with a fixed-frequency peaking filter so users (and the benchmark engine) have a dedicated knob for low-mid character.

## Scoring

Each song has a hidden **benchmark** — the ideal `{ low, mid, high, gain, eq298 }` combination produced by the spectral analysis pipeline (`scripts/benchmark_engine.py`). That benchmark is the reference “green” / perfect setting stored in the database.

When you submit, the server compares your five values to the benchmark:

```
score = 100 − Σ (weight × |your_value − benchmark_value| / 24 × 100)
```

- **100** = exact match with the benchmark (perfect quality).
- Lower scores mean further from the ideal mix.
- Each band is weighted equally (20%) by default.
- The response includes a per-band breakdown (delta in dB and weighted penalty).

Random starting positions (Low / Mid / High / Gain only) ensure every session begins from a different point — users must actively tune toward the best sound rather than starting flat.

## Setup

### Prerequisites

- Node.js 18+
- Python 3.11+ (for benchmark scripts)
- A PostgreSQL database (Supabase free tier works)

### Environment

Copy `.env.example` to `.env` and set:

```env
DATABASE_URL=postgresql://...
AUTH_SECRET=your-long-random-secret
```

### Database

```powershell
py -m pip install -r scripts/requirements.txt
py scripts/setup_db.py --seed-admin
```

Default admin: `admin` / `UNEQWURL-admin` — change before deploying.

### Songs & benchmarks

Place MP3 files in `music/`, then:

```powershell
py scripts/generate_benchmarks.py
```

This probes each track, writes JSON to `benchmark-data/`, copies audio to `public/audio/`, and upserts rows into the `songs` table with `benchmark_ready = true`.

### Run the app

```powershell
npm install
npm run dev          # clears .next cache then starts (avoids stale chunk errors)
npm run dev:fast     # skip cache clear if you need faster restarts
```

### Production build

```powershell
npm run build
npm start
```

## API overview

| Method | Route | Description |
|--------|-------|-------------|
| POST | `/api/auth/register` | Create account |

| POST | `/api/auth/logout` | Clear session |
| GET | `/api/songs` | List benchmark-ready songs |
| POST | `/api/submissions` | Submit EQ settings, get score |
